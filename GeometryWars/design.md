# 几何大战 (Geometry Wars)

基于 GameLib.h 的霓虹风格几何射击游戏，无尽模式，追求爽快手感与视觉冲击力。

## 基本信息

| 项目 | 值 |
|------|-----|
| 窗口尺寸 | 800 x 600 |
| 地图尺寸 | 1200 x 900 |
| 帧率 | 60 FPS |
| 视觉风格 | 黑底 + 背景星空 + 霓虹色几何体 + 粒子爆炸 |
| 渲染方式 | 纯图元绘制，零精灵素材依赖 |

## 游戏流程

无尽模式，不切换场景，所有阶段用 GameLib.h 的 `SetScene` 机制管理：

```
开头画面 → 战斗（持续生成敌人，难度递增） → 玩家死亡（多命） → 最终死亡爆炸清屏 → 总结算画面 → 排行榜 → 开头画面
```

### 场景状态机

使用 `game.SetScene()` 和 `game.GetScene()` 管理游戏大状态：

```
SCENE_TITLE (1) → SCENE_COMBAT (2) → SCENE_DEATH (3) → SCENE_GAME_OVER (4) → SCENE_LEADERBOARD (5) → SCENE_TITLE (1)
                                                          ↑ L键
SCENE_TITLE (1) → SCENE_LEADERBOARD (5) → SCENE_TITLE (1)
```

| 场景 ID | 名称 | 说明 |
|---------|------|------|
| `SCENE_TITLE` (1) | 标题画面 | 显示标题、操作说明、START 按钮（鼠标可点击），Enter/Space 也生效；L 键跳转排行榜 |
| `SCENE_COMBAT` (2) | 战斗 | 正常游戏，敌人生成、射击、计分 |
| `SCENE_DEATH` (3) | 死亡动画 | 播放最终死亡爆炸，约 1.5 秒后自动转入 GAME_OVER |
| `SCENE_GAME_OVER` (4) | 结算画面 | 显示统计和死亡原因，CONTINUE 按钮（鼠标可点击），Space/Enter 也生效 |
| `SCENE_LEADERBOARD` (5) | 排行榜 | 显示 Top 10 高分，CONTINUE 按钮（鼠标可点击），Space/Enter 也生效 |

### 1. 开头画面 (TITLE)

- 背景：黑底 + 弹簧网格缓慢自动波动（无玩家无敌人）。
- 标题：屏幕上部黄金分割线（600 * 0.382 ≈ 229）附近大号 **"GEOMETRY WARS"**（`DrawTextScale`，scale 3~4），霓虹青色，带缓慢脉冲明暗。
- 副标题：标题下方小字 **"PRESS ENTER TO START"**，闪烁。
- **START 按钮**：GameLib `Button`，鼠标可点击，键盘 Enter/Space 也生效。
- 操作说明：按钮下方列出详细操控方式（WASD 移动、鼠标瞄准、左键射击）。
- 历史纪录：画面底部灰色单行显示 `BEST: xxx | TIME M:SS`（历史最高分和最长存活时间）。
- 按 Enter/Space 或点击 START 按钮 → 进入 SCENE_COMBAT，立即开始战斗。
- 按 L → 进入 SCENE_LEADERBOARD，查看排行榜。

### 2. 战斗 (COMBAT)

- 开局直接进入战斗，无预告、无波次间歇。
- 敌人持续不断地从地图四周生成，不会停。
- 早期只有少量简单敌人，随时间推移敌人越来越多、越来越快。
- 所有 4 种敌人类型在游戏开始就陆续出现，不会等很久：
  - 蜂群圆：开局即有
  - 追踪三角：约 15 秒后开始出现
  - 弹跳方块：约 30 秒后开始出现
  - 坦克大圆：约 60 秒后开始出现
- 敌人数量从少到多递增，营造"越来越混乱"的压迫感。
- 无 Boss，无波次，无关卡，纯粹无尽。

### 3. 玩家死亡 → 总结算 (DEATH → GAME_OVER → LEADERBOARD → TITLE)

玩家在 COMBAT 阶段被碰到：

- **多命系统**：玩家有 3 命。非最终死亡时小型爆炸 + 短暂无敌重生（2 秒闪烁无敌），清除附近敌人。
- **最终死亡爆炸清屏**：巨型冲击波清除所有敌人和弹幕，大量白色→彩色粒子，弹簧网格剧烈震荡。
- **总结算画面**：
  - 背景星空 + 网格余波慢慢恢复。
  - 大号 **"GAME OVER"**（红色）。
  - 总分、总击杀数、最高连击倍率、存活时间。
  - **死亡原因**："KILLED BY: [敌人名]"，用敌人对应霓虹色 + 小图标。
  - **CONTINUE 按钮**：GameLib `Button`，鼠标可点击，键盘 Space/Enter 也生效 → 进入排行榜。
- **排行榜画面**：
  - Top 10 高分排行（分数、击杀数、存活时间）。
  - 本次成绩金色高亮。
  - **CONTINUE 按钮**：GameLib `Button`，鼠标可点击，键盘 Space/Enter 也生效 → 回标题画面。

| 快捷键 | 功能 | 使用场景 |

## 摄像机

- 摄像机跟随玩家，玩家始终大致在屏幕中央。
- 摄像机在地图范围内平滑漫游，不会超出地图边界。
- 地图坐标范围：(0, 0) ~ (1200, 900)。

## 地图与边界

- 地图有可见的霓虹色边框围挡（发光边框线）。
- 玩家和敌人均被限制在边界内，不可穿越。
- 子弹飞出边界后消失。

## 弹簧网格背景

核心视觉卖点，铺满整个地图。

## 背景星空

两层缓慢漂移的静态星点粒子，增加画面层次感。

- 远层：50 颗星，小尺寸（1~2px），暗淡（alpha 30~60），缓慢漂移（5~10 px/s 向下），颜色为白/淡蓝/淡青。
- 近层：30 颗星，中等尺寸（2~3px），较亮（alpha 60~100），较快漂移（15~25 px/s 向下）。
- 星点在地图空间移动，随摄像机偏移，漂出地图底部后从顶部重新出现。
- 绘制顺序：星空 → 网格 → 其他所有元素（星空在最底层）。

### 原理

- 将地图划分为网格，每个交叉点是一个质点（有位置和速度）。
- 每个质点通过弹簧力拉回静止位置，同时带阻尼衰减。
- 爆炸和子弹对附近质点施加冲击力，产生涟漪扭曲。
- 用 `DrawLine` 连接相邻质点绘制网格线。

### 参数

| 参数 | 值 |
|------|-----|
| 网格间距 | ~25px（约 50x38 个节点） |
| 网格颜色 | 低亮度蓝紫色，带 Alpha 透明 |
| 弹簧刚度 | 适中，扭曲后 0.3~0.5 秒恢复 |
| 爆炸影响半径 | ~100px |
| 子弹影响半径 | ~40px |

## 玩家

- **外形**：菱形/箭头形状，霓虹青色，朝向指向鼠标方向。
- **移动速度**：中等，手感灵活但不飘。
- **生命**：3 命，被敌人碰到失去 1 命。非最终死亡时短暂无敌重生（2 秒闪烁），清除附近敌人。最终死亡时进入死亡动画。
- **碰撞体**：圆形，半径略小于视觉外形。

## 子弹

- 从玩家位置射向鼠标方向。
- 左键按住连发，射速约每秒 8~10 发。
- 子弹外形：小型发光圆点，带短拖尾。
- 子弹速度：810 px/s，飞行时间短，飞出地图边界后消失。
- 子弹命中敌人或飞出边界时对弹簧网格施加小幅冲击。

## 敌人

4 种几何体，各有不同行为。无尽模式下，所有类型陆续出现，不会等很久。

### 1. 蜂群圆 (Swarm)

- **外形**：小圆，橙色霓虹。
- **行为**：缓慢飘向玩家，成群出现。
- **血量**：1 发击杀。
- **分值**：50 分。
- **特点**：数量多，制造压迫感。
- **首次出现**：开局即有。

### 2. 追踪三角 (Chaser)

- **外形**：三角形，粉红色霓虹，尖端指向移动方向。
- **行为**：快速追踪玩家，转向灵活。
- **血量**：1 发击杀。
- **分值**：100 分。
- **特点**：速度快，威胁大。
- **首次出现**：约 15 秒后。

### 3. 弹跳方块 (Bouncer)

- **外形**：正方形（逆时针旋转 2 圈/秒），绿色霓虹。
- **行为**：匀速直线运动，碰到边界反弹。不追踪玩家。
- **旋转**：每个方块有独立的初始旋转角度，全部逆时针匀速旋转（每秒 2 圈）。虽然移动慢，但旋转使其看起来更灵动。
- **血量**：2 发击杀。
- **分值**：150 分。
- **特点**：轨迹可预测但会切割空间，限制玩家走位。
- **首次出现**：约 30 秒后。

### 4. 坦克大圆 (Tank)

- **外形**：大圆，紫色霓虹，带内环装饰。
- **行为**：缓慢移向玩家。
- **血量**：5 发击杀。
- **分值**：300 分。
- **特点**：被击杀时分裂为 3 个蜂群圆。
- **首次出现**：约 60 秒后。

## 敌人生成

- 敌人生成是持续的过程，不分波次，没有间歇。
- 生成间隔随游戏时间缩短（越来越快）。
- 同一时刻地图上的敌人数量上限随时间增加。
- 敌人从地图四周随机位置进入，确保离玩家有一定距离。

### 生成节奏

| 游戏时间 | 生成间隔 | 同时存在上限 | 可用类型 |
|----------|---------|-------------|---------|
| 0~15 秒 | 0.35s | 15 | 蜂群圆 |
| 15~30 秒 | 0.28s | 20 | 蜂群圆 + 追踪三角 |
| 30~60 秒 | 0.22s | 25 | 蜂群圆 + 追踪三角 + 弹跳方块 |
| 60~120 秒 | 0.22s | 30 | 全部类型 |
| 120 秒+ | 0.18s | 40 | 全部类型 |

## 粒子系统

击杀的视觉冲击力核心。

- **击杀爆炸**：敌人死亡时爆出 15~30 个与敌人同色的粒子，向四周扩散。
- **粒子外形**：发光小圆点，带 Alpha 渐隐。
- **粒子寿命**：约 0.5~1 秒。
- **粒子物理**：初速度快，带摩擦力减速。
- **子弹拖尾**：每帧在子弹位置留下 2~3 个半透明小点。
- **命中火花**：子弹命中敌人时（无论是否击杀），从命中点反弹 5~8 个白色高亮火花，沿子弹反方向散射（±50°），速度 150~400 px/s，寿命 0.3~0.7 秒，尺寸 3~6 像素。在彩色爆炸背景下极其醒目。

## 浮动得分文字

- 敌人死亡时，在命中位置显示黄色 `+得分` 文字（含连击倍率）。
- 文字以 60px/s 速度向上飘动。
- 1 秒生命周期，alpha 渐隐 fade out。
- 同时最多存在 30 个浮动文字。

## 道具系统

### 清屏道具（Nuke）

- **外形**：霓虹青色菱形，带光晕和脉冲呼吸动画。剩余 3 秒时闪烁提示。
- **生成**：每 14~22 秒随机生成一个，位置距离玩家 > 200px，存在 8 秒。
- **触发**：玩家碰到道具即激活。
- **效果**：
  - 全屏敌人立即死亡，每个敌人独立爆炸粒子 + 黄色 `+得分` 飘字。
  - 屏幕强震（8px，20帧）。
  - 弹簧网格强力冲击（半径 600，强度 400）。
  - 玩家位置额外白色大闪光（50 粒子）。
  - 每个敌人都计分（基础分，无连击倍率）。
  - 弹出 "NUKE ACTIVATED!" 通知。

### 能量道具（Energy）

- **外形**：金色菱形，带橙色光晕和脉冲呼吸动画。剩余 3 秒时闪烁提示。
- **掉落**：击杀敌人时概率掉落——蜂群圆 7%、追踪三角 10%、弹跳方块 12%、坦克 17%。活跃敌人数超过 15 时概率递减（每多5个敌人约减10%，最低30%）；100px 内已有能量道具时概率减半。
- **触发**：玩家碰到道具即激活。
- **效果**（持续 5 秒）：
  - 射击变为 5 发扇形散射（角度偏移 -15°、-7°、0°、+7°、+15°）。
  - 射速提升至 0.08s 间隔（约 12.5 次/秒）。
  - 玩家获得金色光晕覆盖。
  - HUD 显示金色能量条（剩余时间比例）。
  - 弹出 "SPREAD SHOT!" 通知。
- **重生时重置**：失去一命时能量效果消失。

## 计分与连击

| 项目 | 规则 |
|------|------|
| 基础分 | 各敌人分值见上方 |
| 连击倍率 | 每次击杀后 2 秒内再击杀，倍率 +1（最高 x10） |
| 倍率衰减 | 2 秒无击杀，倍率重置为 x1 |
| 显示 | 左上角显示分数，倍率用大号字显示 |

## 屏幕文字通知（Achievement Popup）

里程碑达成时屏幕中央弹出大号霓虹文字，配合震动效果。

| 事件 | 条件 | 文字 | 颜色 | 震动 |
|------|------|------|------|------|
| 连击 5x | combo>=5（每局一次） | "x5 COMBO!" | 黄色 | 4px, 10帧 |
| 连击 10x | combo>=10（每局一次） | "x10 COMBO!" | 金色 | 6px, 15帧 |
| 50 杀 | totalKills==50（每局一次） | "50 KILLS!" | 绿色 | 3px, 8帧 |
| 100 杀 | totalKills==100（每局一次） | "100 KILLS!" | 绿色霓虹 | 5px, 12帧 |
| 200 杀 | totalKills==200（每局一次） | "200 KILLS!" | 紫色霓虹 | 7px, 15帧 |
| Nuke 拾取 | triggerNuke 时 | "NUKE ACTIVATED!" | 青色 | （已有震动） |
| Energy 拾取 | 拾取能量道具时 | "SPREAD SHOT!" | 金色 | 3px, 8帧 |

- 弹出动画：0.2s 渐增 → 1.3s 保持 → 0.5s 渐隐（alpha fade out）。
- 每局每种成就只触发一次（防重复）。

## 死亡原因提示

GAME_OVER 画面显示最后一次致死敌人信息。

- 格式："KILLED BY: [敌人名]"，用该敌人的霓虹色显示。
- 左侧附带敌人形状小图标（蜂群圆、追踪三角轮廓、弹跳方块轮廓、坦克大圆+内环）。
- 敌人名：SWARM / CHASER / BOUNCER / TANK。

## 多命系统

- 玩家初始 3 命。
- 被敌人碰到后失去 1 命：
  - **非最终死亡**（lives > 0）：小型爆炸（30+20 粒子）、中等震动（5px, 10帧）、移至地图中央、清除 150px 内敌人、2 秒闪烁无敌、能量效果重置。
  - **最终死亡**（lives = 0）：巨型爆炸、进入 SCENE_DEATH → SCENE_GAME_OVER 流程。
- HUD 左下角显示 "LIVES: 3"。
- 重生期间玩家闪烁（每 8Hz 隐现交替），敌人碰撞无效。

## 高分排行榜

本地 Top 10 分数排行，存档文件 `geometry_wars.sav`。

| 记录项 | 说明 |
|--------|------|
| 排行榜分数 | Top 10 单次游戏最高分 |
| 排行榜时间 | 对应的存活时间 |
| 排行榜击杀 | 对应的击杀数 |
| 排行榜连击 | 对应的最高连击 |

- 排行榜按分数降序排列。
- 本次成绩如入选，该行用金色高亮显示。
- 标题画面按 L 键可查看排行榜。
- 结算画面只能按 Space/Enter 或点击 CONTINUE 按钮进排行榜，再按 Space/Enter 或点击 CONTINUE 按钮回标题。

## 屏幕震动

- 击杀普通敌人时轻微震动（1px，持续 3 帧）。
- 坦克大圆被击杀时中等震动（2px，持续 5 帧）。
- 玩家死亡爆炸时最强震动（8px，持续 20 帧）。

## 技术约束

- 遵守 GameLib.h 的 C++11 / GCC 4.9.2 限制。
- 纯图元绘制（`DrawLine`、`FillCircle`、`FillTriangle`、`FillRect` 等），不依赖精灵素材。
- 编译命令：`g++ -o geometry.exe geometry.cpp -mwindows`
- 文件位置：`games/geometry/geometry.cpp`，通过 `#include "../../GameLib.h"` 引入。
