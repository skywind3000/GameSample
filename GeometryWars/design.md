# 几何大战 (Geometry Wars)

基于 GameLib.h 的霓虹风格几何射击游戏，无尽模式，追求爽快手感与视觉冲击力。

## 基本信息

| 项目 | 值 |
|------|-----|
| 窗口尺寸 | 800 x 600 |
| 地图尺寸 | 1200 x 900 |
| 帧率 | 60 FPS |
| 视觉风格 | 黑底 + 霓虹色几何体 + 粒子爆炸 |
| 渲染方式 | 纯图元绘制，零精灵素材依赖 |

## 游戏流程

无尽模式，不切换场景，所有阶段用 GameLib.h 的 `SetScene` 机制管理：

```
开头画面 → 战斗（持续生成敌人，难度递增） → 玩家死亡 → 死亡爆炸清屏 → 总结算画面 → 开头画面
```

### 场景状态机

使用 `game.SetScene()` 和 `game.GetScene()` 管理游戏大状态：

```
SCENE_TITLE (1) → SCENE_COMBAT (2) → SCENE_DEATH (3) → SCENE_GAME_OVER (4) → SCENE_TITLE (1)
```

| 场景 ID | 名称 | 说明 |
|---------|------|------|
| `SCENE_TITLE` (1) | 标题画面 | 显示标题、操作说明，等待 Enter |
| `SCENE_COMBAT` (2) | 战斗 | 正常游戏，敌人生成、射击、计分 |
| `SCENE_DEATH` (3) | 死亡动画 | 播放死亡爆炸，约 1.5 秒后自动转入 GAME_OVER |
| `SCENE_GAME_OVER` (4) | 结算画面 | 显示统计，按 R 回到 TITLE |

### 1. 开头画面 (TITLE)

- 背景：黑底 + 弹簧网格缓慢自动波动（无玩家无敌人）。
- 标题：屏幕中央大号 **"GEOMETRY WARS"**（`DrawTextScale`，scale 3~4），霓虹青色，带缓慢脉冲明暗。
- 副标题：标题下方小字 **"PRESS ENTER TO START"**，闪烁。
- 操作说明：标题下方列出详细操控方式（WASD 移动、鼠标瞄准、左键射击、Enter 开始）。
- 按 Enter → 进入 SCENE_COMBAT，立即开始战斗。

### 2. 战斗 (COMBAT)

- 开局直接进入战斗，无预告、无波次间歇。
- 敌人持续不断地从地图四周生成，不会停。
- 早期只有少量简单敌人，随时间推移敌人越来越多、越来越快。
- 所有 4 种敌人类型在游戏开始就陆续出现，不会等很久：
  - 蜂群圆：开局即有
  - 追踪三角：约 15 秒后开始出现
  - 弹跳方块：约 30 秒后开始出现
  - 坦克大圆：约 60 秒后开始出现
- 敌人数量从少到多递增，营造"越来越混乱"的压迫感。
- 无 Boss，无波次，无关卡，纯粹无尽。

### 3. 玩家死亡 → 总结算 (DEATH → GAME_OVER)

玩家在 COMBAT 阶段被碰到：

- **死亡爆炸清屏**：巨型冲击波清除所有敌人和弹幕，大量白色→彩色粒子，弹簧网格剧烈震荡。
- **总结算画面**：
  - 背景全黑 + 网格余波慢慢恢复。
  - 大号 **"GAME OVER"**（红色）。
  - 总分、总击杀数、最高连击倍率、存活时间。
  - **"PRESS R TO RESTART"** → 回到 TITLE。

## 操控

| 操作 | 方式 |
|------|------|
| 移动 | WASD 四方向（可对角线组合） |
| 瞄准 | 鼠标位置，玩家朝向始终指向鼠标 |
| 射击 | 鼠标左键按住连发 |
| 开始 | Enter（开头画面） |
| 重来 | R（总结算画面） |
| **F9** | 调试：无敌切换（开启后不会被敌人/弹幕伤害，再按关闭） |

### 调试快捷键说明

| 快捷键 | 功能 | 使用场景 |
|--------|------|---------|
| F9 | 切换无敌模式 | 测试战斗流程、避免死亡中断测试 |

## 摄像机

- 摄像机跟随玩家，玩家始终大致在屏幕中央。
- 摄像机在地图范围内平滑漫游，不会超出地图边界。
- 地图坐标范围：(0, 0) ~ (1200, 900)。

## 地图与边界

- 地图有可见的霓虹色边框围挡（发光边框线）。
- 玩家和敌人均被限制在边界内，不可穿越。
- 子弹飞出边界后消失。

## 弹簧网格背景

核心视觉卖点，铺满整个地图。

### 原理

- 将地图划分为网格，每个交叉点是一个质点（有位置和速度）。
- 每个质点通过弹簧力拉回静止位置，同时带阻尼衰减。
- 爆炸和子弹对附近质点施加冲击力，产生涟漪扭曲。
- 用 `DrawLine` 连接相邻质点绘制网格线。

### 参数

| 参数 | 值 |
|------|-----|
| 网格间距 | ~25px（约 50x38 个节点） |
| 网格颜色 | 低亮度蓝紫色，带 Alpha 透明 |
| 弹簧刚度 | 适中，扭曲后 0.3~0.5 秒恢复 |
| 爆炸影响半径 | ~100px |
| 子弹影响半径 | ~40px |

## 玩家

- **外形**：菱形/箭头形状，霓虹青色，朝向指向鼠标方向。
- **移动速度**：中等，手感灵活但不飘。
- **生命**：一命通关，被任何敌人碰到即死亡。
- **碰撞体**：圆形，半径略小于视觉外形。

## 子弹

- 从玩家位置射向鼠标方向。
- 左键按住连发，射速约每秒 8~10 发。
- 子弹外形：小型发光圆点，带短拖尾。
- 子弹速度：810 px/s，飞行时间短，飞出地图边界后消失。
- 子弹命中敌人或飞出边界时对弹簧网格施加小幅冲击。

## 敌人

4 种几何体，各有不同行为。无尽模式下，所有类型陆续出现，不会等很久。

### 1. 蜂群圆 (Swarm)

- **外形**：小圆，橙色霓虹。
- **行为**：缓慢飘向玩家，成群出现。
- **血量**：1 发击杀。
- **分值**：50 分。
- **特点**：数量多，制造压迫感。
- **首次出现**：开局即有。

### 2. 追踪三角 (Chaser)

- **外形**：三角形，粉红色霓虹，尖端指向移动方向。
- **行为**：快速追踪玩家，转向灵活。
- **血量**：1 发击杀。
- **分值**：100 分。
- **特点**：速度快，威胁大。
- **首次出现**：约 15 秒后。

### 3. 弹跳方块 (Bouncer)

- **外形**：正方形（旋转 45° 呈菱形），绿色霓虹。
- **行为**：匀速直线运动，碰到边界反弹。不追踪玩家。
- **血量**：2 发击杀。
- **分值**：150 分。
- **特点**：轨迹可预测但会切割空间，限制玩家走位。
- **首次出现**：约 30 秒后。

### 4. 坦克大圆 (Tank)

- **外形**：大圆，紫色霓虹，带内环装饰。
- **行为**：缓慢移向玩家。
- **血量**：5 发击杀。
- **分值**：300 分。
- **特点**：被击杀时分裂为 3 个蜂群圆。
- **首次出现**：约 60 秒后。

## 敌人生成

- 敌人生成是持续的过程，不分波次，没有间歇。
- 生成间隔随游戏时间缩短（越来越快）。
- 同一时刻地图上的敌人数量上限随时间增加。
- 敌人从地图四周随机位置进入，确保离玩家有一定距离。

### 生成节奏

| 游戏时间 | 生成间隔 | 同时存在上限 | 可用类型 |
|----------|---------|-------------|---------|
| 0~15 秒 | 0.35s | 15 | 蜂群圆 |
| 15~30 秒 | 0.28s | 20 | 蜂群圆 + 追踪三角 |
| 30~60 秒 | 0.22s | 25 | 蜂群圆 + 追踪三角 + 弹跳方块 |
| 60~120 秒 | 0.22s | 30 | 全部类型 |
| 120 秒+ | 0.18s | 40 | 全部类型 |

## 粒子系统

击杀的视觉冲击力核心。

- **击杀爆炸**：敌人死亡时爆出 15~30 个与敌人同色的粒子，向四周扩散。
- **粒子外形**：发光小圆点，带 Alpha 渐隐。
- **粒子寿命**：约 0.5~1 秒。
- **粒子物理**：初速度快，带摩擦力减速。
- **子弹拖尾**：每帧在子弹位置留下 2~3 个半透明小点。

## 计分与连击

| 项目 | 规则 |
|------|------|
| 基础分 | 各敌人分值见上方 |
| 连击倍率 | 每次击杀后 2 秒内再击杀，倍率 +1（最高 x10） |
| 倍率衰减 | 2 秒无击杀，倍率重置为 x1 |
| 显示 | 左上角显示分数，倍率用大号字显示 |

## 屏幕震动

- 击杀普通敌人时轻微震动（1px，持续 3 帧）。
- 坦克大圆被击杀时中等震动（2px，持续 5 帧）。
- 玩家死亡爆炸时最强震动（8px，持续 20 帧）。

## 技术约束

- 遵守 GameLib.h 的 C++11 / GCC 4.9.2 限制。
- 纯图元绘制（`DrawLine`、`FillCircle`、`FillTriangle`、`FillRect` 等），不依赖精灵素材。
- 编译命令：`g++ -o geometry.exe geometry.cpp -mwindows`
- 文件位置：`games/geometry/geometry.cpp`，通过 `#include "../../GameLib.h"` 引入。
